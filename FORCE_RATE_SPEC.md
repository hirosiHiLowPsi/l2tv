# L2TV FORCE RATE 仕様書

この文書は、L2TV における FORCE RATE の計算仕様をまとめたものです。

## 1. 概要

FORCE RATE は、発狂BMS難易度表および第二期Overjoy難易度表を対象に、プレイヤーの高難度譜面に対する総合到達度を数値化するレーティングです。

基本単位は単曲ごとの `Chart FORCE` で、これを上位譜面と段位コースから集計してプレイヤーの `FORCE RATE` を算出します。

## 2. 対象譜面

対象は以下の難易度表に登録されている譜面です。

- 発狂BMS難易度表
- 第二期Overjoy難易度表

第二期Overjoyは `https://lr2.sakura.ne.jp/overjoy.php` を参照元とします。

第三期Overjoyのみに登録されている譜面は対象外です。

発狂BMS難易度表と第二期Overjoyの両方に存在する譜面は、発狂BMS難易度表側の譜面として扱います。

第二期Overjoyの表記レベルは、内部的に次のように発狂BMS難易度へ換算します。

```text
★★n = ★(20 + n)
```

例:

```text
★★0 = ★20
★★5 = ★25
```

## 3. 対象外

次の譜面・記録は FORCE RATE の対象外です。

- NO PLAY
- NO SONG
- MD5で照合できない譜面
- FORCE譜面定数表に存在しない譜面
- 第三期Overjoyのみに存在する譜面
- 不正者・CHEATED扱いのスコア

## 4. 譜面定数

譜面定数は、LR2IR Archive の記録をもとに算出した固定値です。

アプリでは次のJSONを読み込みます。

```text
public/data/force-chart-constants.json
```

主な項目は以下です。

| 項目 | 内容 |
|---|---|
| `md5` | 譜面MD5 |
| `title` | 曲名 |
| `chartConstant` | FORCE RATEで使う最終譜面定数 |
| `sourceTable` / `source` | 所属難易度表 |
| `difficulty` | 難易度表上の表記 |
| `nominalLevel` | 換算後の表記レベル |

## 5. 譜面定数の決め方

譜面定数は、単純な表記レベルではなく、LR2IR Archive上のクリア状況とスコア状況から推定します。

### 5.1 対象帯

譜面を以下の帯に分けて扱います。

| 帯 | レベル |
|---|---|
| 低難度 | ★1～★8 |
| 中難度 | ★9～★16 |
| 高難度 | ★17～★27 |
| 到達不能 | ★28 |

低・中・高難度では、その帯に属する譜面の50%以上をプレイしているプレイヤーを候補にし、各帯ごとに最大2000人をサンプルとして扱います。

### 5.2 IRT推定

クリアランプを順序尺度として扱い、IRTで譜面難度を推定します。

順序は以下です。

```text
FAILED < EASY CLEAR < NORMAL CLEAR < HARD CLEAR < FULL COMBO
```

### 5.3 スコア重視版の譜面定数

譜面定数は、難易度表の表記レベルを基準にしつつ、LR2IR Archive上のスコア分布を主軸として補正します。

旧来のランプIRT推定は、最終定数を決める主役ではなく、ランプ状況から見た難しさを軽く反映する補助値として扱います。

```text
最終譜面定数 = roundTo2(clamp(表記レベル + スコア乖離 + ランプIRT乖離, 1.00, 27.00))
```

★28以上は到達不能譜面として、最終譜面定数を `27.00` に固定します。

```text
表記レベル >= 28 の場合:
  最終譜面定数 = 27.00
```

### 5.4 スコア分布補正

スコア分布補正では、対象帯の50%以上をプレイしているプレイヤーをサンプルにし、その譜面で有効なEX SCOREを持つプレイヤーを母数にします。

母数に含める記録:

- EX SCOREが存在する
- MAX EX SCOREが0より大きい
- CHEATED扱いではない
- 不正者として除外されていない

FAILEDでも、有効なEX SCOREが存在する場合は母数に含めます。

スコア閾値は以下です。

| 閾値 | 基準到達率 | 係数 | 最小補正 | 最大補正 |
|---|---:|---:|---:|---:|
| AAA / 88.89% | 0.45 | 0.18 | -0.30 | +0.45 |
| 94.44% | 0.20 | 0.35 | -0.50 | +1.20 |
| 97.00% | 0.08 | 0.45 | -0.40 | +1.40 |
| 99.00% | 0.02 | 0.55 | -0.30 | +1.60 |

各閾値について、到達者が10人未満の場合は補正を0にします。

```text
到達者数 < 10 の場合:
  閾値補正 = 0
```

到達者が10人以上いる場合は、到達率を平滑化して補正します。

```text
到達率 = (到達者数 + 1) / (有効スコア人数 + 2)
生補正 = clamp(log2(基準到達率 / 到達率) × 係数, 最小補正, 最大補正)
信頼度 = clamp(有効スコア人数 / 100, 0, 1)
閾値補正 = 生補正 × 信頼度
```

スコア乖離は、各閾値補正の合計を次の範囲に収めます。

```text
スコア乖離 = clamp(AAA補正 + 94.44補正 + 97補正 + 99補正, -1.00, +3.00)
```

### 5.5 ランプIRT補助

ランプIRTは、クリアランプ状況から見た難しさを弱めに反映する補助値です。

```text
ランプIRT乖離 = clamp((ランプIRT定数 - 表記レベル) × 0.35, -1.00, +1.00)
```

この補正は、スコア分布だけでは拾いにくいクリア難度の差を少量だけ残す目的で使います。

### 5.6 最終譜面定数

```text
最終譜面定数 = roundTo2(clamp(表記レベル + スコア乖離 + ランプIRT乖離, 1.00, 27.00))
```

アプリでは小数第2位までを主な表示値として扱います。定数JSONには `constantVersion: "score-oriented-v2"` として、各閾値の母数・到達者数・補正値も保存します。

配布版の `public/data/force-chart-constants.json` は軽量化のため、各譜面を `md5` / `chartConstant` / `source` の最小項目に絞ります。閾値別の詳細な計算結果は、生成時に `outputs/force-score-oriented-constants-YYYYMMDD/force-chart-constants-score-oriented-audit.json` へ保存します。

## 6. Chart FORCE

単曲ごとのレートを `Chart FORCE` と呼びます。

```text
Chart FORCE = 譜面定数 × スコア係数 × クリアランプ係数
```

## 7. スコア係数

スコア係数は、EX SCORE RATE を0～1に換算した値です。

```text
スコア係数 = round((EX SCORE / MAX EX SCORE), 3)
```

例:

| EX SCORE RATE | スコア係数 |
|---:|---:|
| 100.00% | 1.000 |
| 99.00% | 0.990 |
| 94.44% | 0.944 |
| 93.53% | 0.935 |
| 90.00% | 0.900 |
| AAA境界 | 0.889 |

## 8. クリアランプ係数

| ランプ | 係数 |
|---|---:|
| FULL COMBO | 1.02 |
| HARD CLEAR | 0.98 |
| NORMAL CLEAR | 0.93 |
| EASY CLEAR | 0.86 |
| FAILED | 0.50 |

NO PLAY と NO SONG は対象外です。

## 9. 段位加算

GENOSIDE2018 SP段位認定の最高合格段位を、単曲50譜面とは別に1件だけ追加対象にします。

段位は「コース自体に合格していること」を条件とし、段位構成曲の単曲クリア状況では判定しません。

段位ランプ係数:

| 段位ランプ | 係数 |
|---|---:|
| HARD CLEAR | 1.00 |
| NORMAL CLEAR | 0.98 |

ただし、GENOSIDE2018 Overjoy は現実的にHARD CLEAR達成者が存在しないため、NORMAL CLEAR と HARD CLEAR を同等に扱います。

| Overjoy段位ランプ | 係数 |
|---|---:|
| HARD CLEAR | 1.00 |
| NORMAL CLEAR | 1.00 |

段位にはスコア係数を使用しません。

段位定数:

| 段位 | 定数 |
|---|---:|
| 発狂初段 | 1.00 |
| 発狂二段 | 1.00 |
| 発狂三段 | 1.00 |
| 発狂四段 | 1.00 |
| 発狂五段 | 1.00 |
| 発狂六段 | 1.39 |
| 発狂七段 | 1.76 |
| 発狂八段 | 2.94 |
| 発狂九段 | 7.09 |
| 発狂十段 | 12.20 |
| 発狂皆伝 | 18.15 |
| Overjoy | 26.81 |

段位FORCE:

```text
段位FORCE = 段位定数 × 段位ランプ係数
```

## 10. FORCE RATE

まず、Chart FORCE上位50譜面を選びます。

その後、合格済み最高段位が存在する場合は、段位FORCEを1件追加します。

```text
対象 = Chart FORCE上位50譜面 + 最高合格段位1件
```

段位が存在する場合、対象数は最大51件です。

FORCE RATEは次の式で計算します。

```text
全対象平均 = 対象FORCE合計 / 対象数
BEST20平均 = 対象FORCE上位20件の平均

FORCE RATE = 全対象平均 × 0.2 + BEST20平均 × 0.8
```

ただし、BEST20が20件未満の場合は、BEST20平均の代わりに全対象平均を使用します。

表示上限は `30.000` です。

```text
表示FORCE RATE = clamp(FORCE RATE, 0, 30)
```

## 12. 表示項目

Player Data には以下を表示します。

- FORCE RATE
- 称号
- BEST20 / 対象数
- 譜面数 50/50
- FORCE対象譜面数

FORCE RATE TARGETS フォルダには、対象譜面・段位を一覧表示します。

主な列:

- #
- Title
- 譜面定数
- Lamp
- EX/Rate
- 単曲レート
- 対象表

## 13. 実装上の注意

定数生成は次のコマンドで行います。

```text
npm run force:constants:score-oriented
```

既定では、リポジトリの1つ上の階層にある `lr2ir-archive.db` を参照します。別の場所にDBを置く場合は、スクリプトへDBパスを渡します。

```text
node scripts/apply-score-oriented-force-constants.js C:\path\to\lr2ir-archive.db
```

DBが空、または `pb` / `user` / `chart` テーブルが存在しない場合は、古い定数へフォールバックせずにエラーで停止します。
